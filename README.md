## 🌟 Overview

TypeBlitz is a **full-stack, production-grade typing game platform** built for India's most serious typists — SSC/UPSC/Banking/Railway exam aspirants, software developers, and competitive typists who demand more than just a WPM counter.

It combines **arcade-style games** with **structured professional training**, tracks every keystroke error to build a **letter-accuracy heatmap**, and surfaces it all through a **real-time personal dashboard**.

> *"Every keystroke has weight. Train like an athlete, compete like a gamer."*

## 🏗 Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                     Reverse Proxy  ( :80 )                       │
│      /api  →  API Server  (8080)   |   /  →  Frontend (Vite)    │
└───────────────────────┬─────────────────────────────────────────┘
                        │
         ┌──────────────┴───────────────┐
         │                              │
┌────────▼────────┐           ┌─────────▼──────────┐
│   Frontend      │  HTTP     │    API Server        │
│  React + Vite   │◄─────────►│   Express 5          │
│  Port: $PORT    │  REST     │   Port: 8080         │
└────────┬────────┘           └──────────┬──────────┘
         │                               │
         │  Generated hooks              │  Drizzle ORM
         │  (Orval + React Query)        │
┌────────▼────────┐           ┌──────────▼──────────┐
│  lib/api-spec   │           │     PostgreSQL        │
│  OpenAPI 3.0    │           │   users              │
│  (source truth) │           │   sessions           │
└─────────────────┘           │   letterStats        │
                              └─────────────────────-┘
```

### Contract-First API Design
```
1. Edit  lib/api-spec/openapi.yaml          ← single source of truth
2. Run   pnpm --filter @workspace/api-spec run codegen
3. Gets  lib/api-client-react/src/generated/api.ts  ← React Query hooks + Zod
4. Both  frontend (hooks) and server (Zod validation) share the same contract
```

### Authentication Flow
```
Register  POST /api/auth/register  ──►  scrypt hash  ──►  stored in DB
Login     POST /api/auth/login     ──►  verify hash  ──►  HMAC Bearer token
Requests  Authorization: Bearer <token>  ──►  customFetch (AuthContext)
                                        ──►  token verified server-side
```

## 🚀 Getting Started

### Prerequisites
- Node.js 20 or later
- pnpm 9 or later
- PostgreSQL database (local or hosted)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/yourusername/typeblitz.git
cd typeblitz

# 2. Install all workspace dependencies
pnpm install

# 3. Set environment variables
cp .env.example .env
# Edit .env with your DATABASE_URL and SESSION_SECRET

# 4. Push database schema to your PostgreSQL instance
pnpm --filter @workspace/db run push

# 5. Start the API server  (terminal 1)
pnpm --filter @workspace/api-server run dev

# 6. Start the frontend   (terminal 2)
pnpm --filter @workspace/typeblitz run dev
```

### Development Commands

```bash
# Full typecheck across all packages
pnpm run typecheck

# Regenerate API hooks from OpenAPI spec (run after any spec change)
pnpm --filter @workspace/api-spec run codegen

# Typecheck individual packages
pnpm --filter @workspace/typeblitz run typecheck
pnpm --filter @workspace/api-server run typecheck

# Build all packages
pnpm run build

# Push DB schema changes (dev only)
pnpm --filter @workspace/db run push
```

> ⚠️ **Never run `pnpm run dev` at workspace root.** Use per-package commands above.

## 📡 API Reference

Base URL: `/api`

### Auth
| Method | Endpoint | Body / Params | Description |
|--------|----------|---------------|-------------|
| `POST` | `/auth/register` | `{ username, password }` | Create account |
| `POST` | `/auth/login` | `{ username, password }` | Returns Bearer token |
| `GET` | `/auth/me` | `Authorization: Bearer` | Current user profile |

### Games
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/games` | List all 11 games with levels |
| `GET` | `/games/:id` | Single game details |
| `GET` | `/games/:id/levels/:n/words` | Words/text for a specific level |

### Sessions & Progress
| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/sessions` | Save a completed typing session |
| `GET` | `/sessions?userId=` | User's full session history |
| `GET` | `/sessions/stats?userId=` | Aggregated WPM trend & accuracy stats |
| `POST` | `/sessions/analyze` | Analyze custom text for letter-level errors |

### Leaderboard
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/leaderboard` | Global best WPM per player |
| `GET` | `/leaderboard?gameId=` | Per-game filtered leaderboard |

## 📚 Vocabulary Coverage

### Government Exam Vocabulary

TypeBlitz ships with hand-curated vocabulary for all major Indian government competitive exams:

```
SSC CGL / CHSL      — synonyms, antonyms, one-word substitution
UPSC Civil Services — administrative, constitutional, polity terms
```

## 📐 WPM Calculation Methodology

TypeBlitz uses the **industry-standard WPM formula**:

```
WPM  =  (Correct Characters ÷ 5)  ÷  Minutes Elapsed
```

### Key Principles

| Rule | Description |
|------|-------------|
| ✅ Correct chars only | Only correctly typed characters count |
| ❌ Errors don't inflate | Mistyped characters do NOT increase WPM |
| ✅ Real-time | WPM updates every 800ms as you type |
| ✅ Ref-based accuracy | Uses `useRef` so interval always reads latest char count |
| ✅ Timer on first key | Timer starts on first keystroke, not page load |

### Accuracy Formula

```
Accuracy  =  (Correct Characters ÷ Total Characters Typed)  × 100
```

**Why this matters:** Many typing platforms count raw keystrokes including errors, inflating scores. TypeBlitz uses the same formula as professional typing certifications — your WPM is honest.

## 🤝 Contributing

```bash
# 1. Fork and clone
git clone https://github.com/yourusername/typeblitz.git

# 2. Create a feature branch
git checkout -b feature/daily-challenge

# 3. Make your changes, then typecheck
pnpm run typecheck

# 4. If you changed the OpenAPI spec, regenerate hooks
pnpm --filter @workspace/api-spec run codegen

# 5. Commit with conventional commits
git commit -m "feat: add daily challenge mode"

# 6. Open a pull request
```

**Adding vocabulary:** Edit `artifacts/api-server/src/data/words.ts` — each game key maps to an array of 5 level arrays.

**Adding a new game:**
1. Add definition to `artifacts/api-server/src/data/games.ts`
2. Add word lists to `artifacts/api-server/src/data/words.ts`
3. Add theme + icon entry to `artifacts/typeblitz/src/pages/games.tsx`
4. If it's an arcade-style game, add it to `ARCADE_GAME_IDS` in `play.tsx` and create a visual component
